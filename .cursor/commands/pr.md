为当前更改提交并推送，根据分支自动决定是否创建 Pull Request。

1. 使用 `git diff` 查看已暂存和未暂存的更改
2. 根据更改内容编写清晰的提交信息
3. 检查当前分支：
   - 如果在 `main` 分支：提交并直接推送到 `main`，不创建 PR
   - 如果在功能分支：提交并推送到功能分支，然后使用 `gh pr create` 创建 Pull Request（从功能分支到 `main`），并提供标题和描述
4. 如果在功能分支，完成后返回 PR URL；如果在 `main` 分支，返回推送成功信息
